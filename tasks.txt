Сабджект:

+	Вы должны использовать miniLibX. Либо версия, доступная в операционной системе, либо из ее источников. Если вы решите работать с источниками, вам нужно будет применить те же правила для вашего libft, что и те, которые написаны выше в части общих инструкций.
+	Управление вашим окном должно оставаться плавным: переход в другое окно, сворачивание и т.д.
+/-	Вам нужны по крайней мере эти 3 простых геометрических объекта: плоскость, сфера, цилиндр.
+	Если применимо, все возможные пересечения и внутренняя часть объекта должны быть обработаны правильно.
-	Ваша программа должна иметь возможность изменять уникальные свойства объекта: диаметр для сферы и ширину и высоту для цилиндра.
-	Ваша программа должна иметь возможность применять преобразование перемещения и поворота к объектам, источникам света и камерам (за исключением сфер и источников света, которые нельзя поворачивать).
+/-	Управление освещением: точечная яркость, резкие тени, окружающее освещение (объекты никогда не находятся полностью в темноте). Вы должны использовать рассеянное и рассеянное освещение.

+	Нажатие клавиши ESC должно закрыть окно и завершить работу программы.
+	Нажатие на красный крестик в рамке окна должно закрыть окно и полностью выйти из программы.

+	Настоятельно рекомендуется использовать изображения minilibX.

+	Ваша программа должна принимать в качестве первого аргумента файл описания сцены с расширением .rt.
+	Каждый тип элемента может быть разделен одним или несколькими разрывами строк.
+	Каждый тип информации из элемента может быть разделен одним или несколькими пробелами.
+	Каждый тип элемента может быть установлен в любом порядке в файле.
+	Элементы, которые обозначаются заглавной буквой, могут быть объявлены в сцене только один раз.
+	Информация о каждом элементе сначала представляет собой идентификатор типа (состоящий из одного или двух символов), за которым следует вся конкретная информация для каждого объекта в строгом порядке, например:

+	Если в файле обнаружена какая-либо неправильная конфигурация любого рода, программа должна завершиться должным образом и вернуть "Error \n", за которой следует явное сообщение об ошибке по вашему выбору.
-	Для защиты было бы идеально, если бы у вас был целый набор сцен с акцентом на то, что функционально, чтобы облегчить контроль над создаваемыми элементами.
